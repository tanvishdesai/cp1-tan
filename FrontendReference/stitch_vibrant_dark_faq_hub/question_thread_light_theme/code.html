Why is my React component re-rendering infinitely when updating state inside useEffect?

Alex Chen
Asked 2 days ago • 1.2k views

I'm trying to fetch some data based on a prop change, but I seem to have created an infinite loop. The browser tab freezes completely when I navigate to this view.

Here is the simplified version of my component:

import React, { useState, useEffect } from 'react';

function DataDisplay({ userId }) {
  const [data, setData] = useState([]);
  const [filters, setFilters] = useState({ active: true });

  useEffect(() => {
    fetchData(userId, filters).then(result => {
      setData(result);
      // I think this next line might be the issue?
      setFilters({ ...filters, lastFetched: Date.now() }); 
    });
  }, [userId, filters]); // Added filters to dependencies because ESLint complained

  return (
    <div>{/* rendering logic */}</div>
  );
}

I understand that updating `filters` causes a re-render, which triggers `useEffect` again because `filters` is in the dependency array. But if I remove `filters` from the array, ESLint gives me a `react-hooks/exhaustive-deps` warning.

What is the elegant, "React way" to handle this without disabling the linter rule?

The core issue here is indeed the cyclical dependency. You are fetching data, updating filters, which triggers the effect again because filters is a new object reference (or its values changed), leading to another fetch.

The fundamental rule is: If you only want to fetch when userId changes, do not update state that is in the dependency array from within that same effect.

You have a couple of architectural choices to fix this cleanly:

1. Separate the concerns (Recommended)

Ask yourself: why does lastFetched need to be part of the filters state? Usually, filters dictate what to fetch, not metadata about when it was fetched. Separate them:

function DataDisplay({ userId }) {
  const [data, setData] = useState([]);
  const [filters, setFilters] = useState({ active: true });
  const [lastFetched, setLastFetched] = useState(null); // Separate state

  useEffect(() => {
    let isMounted = true; // Cleanup to prevent state updates if unmounted

    fetchData(userId, filters).then(result => {
      if (isMounted) {
        setData(result);
        setLastFetched(Date.now()); // Update metadata separately
      }
    });

    return () => { isMounted = false; };
  }, [userId, filters]); // Effect runs when user OR filters change

  // ...
}

Now, updating lastFetched doesn't trigger the effect because it's not in the dependency array (and shouldn't be).

If you absolutely must keep lastFetched inside the filters object for some reason (e.g., passing a single object down to deeply nested children), you can use a ref to store mutable data that doesn't trigger a re-render.

const lastFetchedRef = useRef(null);

useEffect(() => {
  fetchData(userId, filters).then(result => {
    setData(result);
    lastFetchedRef.current = Date.now(); // Does not cause re-render
  });
}, [userId, filters]);

However, Sarah's answer above separating the state is generally the cleaner React paradigm.
